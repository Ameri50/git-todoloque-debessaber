Guía Básica de Git

Comandos Básicos

  • git init
    Inicializa un nuevo repositorio Git en el directorio actual.
    Dificultades: Asegúrate de estar en el directorio correcto; de lo contrario, puedes crear un repositorio en el lugar equivocado.
  • git clone <url>
    Clona un repositorio remoto en tu máquina local.
    Dificultades: Problemas de conexión a Internet o URL incorrecta pueden causar fallos en la clonación.
  • git status
    Muestra el estado actual del repositorio.
    Dificultades: Puede ser confuso si hay muchos cambios; es clave entender lo que se está rastreando.
  • git add <archivo>
    Agrega cambios de un archivo específico al área de preparación.
    Dificultades: Olvidar agregar archivos puede causar que no se incluyan en el commit.
  • git commit -m "Mensaje"
    Crea un nuevo commit con los cambios.
    Dificultades: Mensajes poco descriptivos pueden dificultar la comprensión del historial más tarde.

Ramas

  • git branch
    Lista las ramas en el repositorio actual.
    Dificultades: No ver las ramas esperadas puede indicar que no se han creado correctamente.
  • git branch <nombre-de-la-rama>
    Crea una nueva rama con el nombre especificado.
    Dificultades: Nombres de ramas duplicados causarán errores.
  • git checkout <nombre-de-la-rama>
    Cambia a la rama especificada.
    Dificultades: No poder cambiar de rama si hay cambios sin confirmar en la rama actual.
  • git merge <nombre-de-la-rama>
    Mergia la rama especificada en la rama actual.
    Dificultades: Conflictos de fusión que pueden surgir si los cambios en ambas ramas afectan las mismas líneas.

Repositorios Remotos

  • git remote -v
    Muestra los repositorios remotos configurados.
    Dificultades: Si no ves el remoto esperado, puede que no se haya configurado correctamente.
  • git push
    Envía los commits locales a un repositorio remoto.
    Dificultades: Problemas de autenticación pueden bloquear el envío.
  • git pull
    Descarga y fusiona cambios de un repositorio remoto.
    Dificultades: Conflictos similares a los conflictos de fusión locales.

Cambios

  • git reset HEAD <archivo>
    Deshace los cambios en el área de preparación para un archivo específico.
    Dificultades: Podrías perder cambios si no se usa correctamente.
  • git revert <commit-hash>
    Revierte un commit específico.
    Dificultades: Puede ser confuso entender cómo afecta a la historia del proyecto.
